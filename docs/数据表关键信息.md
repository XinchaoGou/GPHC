# 数据表关键信息

### 1. brands

用于存储德国补剂品牌。

**关键字段：**

- `id`：品牌唯一 ID
- `name`：品牌名称，例如 Doppelherz
- `slug`：URL 标识，例如 doppelherz
- `country`：国家，例如 Germany
- `positioning`：品牌定位，例如大众药房品牌、高端药房品牌、dm 自有品牌
- `official_url`：官网链接
- `description`：品牌简介

### 2. products

用于存储具体补剂产品。

**关键字段：**

- `id`：产品唯一 ID
- `brand_id`：关联 brands.id
- `name_cn`：中文产品名
- `name_original`：德文/原文产品名
- `slug`：URL 标识
- `category`：产品大类，例如 Vitamins、Minerals、Omega-3、Sleep Support
- `dosage_form`：剂型，例如 tablet、capsule、powder、liquid
- `package_size`：规格，例如 60 tablets
- `image_url`：产品图片
- `summary`：产品简介
- `warnings`：产品注意事项

### 3. ingredients

用于存储标准化成分信息。

**关键字段：**

- `id`：成分唯一 ID
- `name_cn`：中文名，例如 镁
- `name_en`：英文名，例如 Magnesium
- `name_de`：德文名，例如 Magnesium
- `slug`：URL 标识
- `category`：成分类别，例如 vitamin、mineral、fatty acid
- `food_sources`：食物来源
- `summary`：成分简介
- `function_summary`：主要作用
- `warnings`：风险与注意事项
- `evidence_level`：证据等级，例如 strong、moderate、limited

### 4. product_ingredients

用于连接 products 和 ingredients，是核心关联表。一个产品可以含有多个成分，一个成分也可以出现在多个产品中。

**关键字段：**

- `id`：唯一 ID
- `product_id`：关联 products.id
- `ingredient_id`：关联 ingredients.id
- `amount`：剂量数值
- `unit`：单位，例如 mg、µg、IU
- `serving_size`：剂量对应的服用单位，例如 1 tablet、2 capsules
- `ingredient_form`：成分形式，例如 citrate、oxide、gluconate
- `daily_value_percent`：每日参考摄入百分比
- `note`：备注

### 5. health_topics

用于存储健康主题页面。

**关键字段：**

- `id`：主题唯一 ID
- `title`：主题名称，例如 睡眠、心血管健康、眼部健康
- `slug`：URL 标识，例如 sleep、cardiovascular-health
- `summary`：主题简介
- `lifestyle_advice`：生活方式建议
- `supplement_notes`：补剂相关说明
- `warning_notes`：风险提醒
- `medical_boundary`：医疗边界说明，例如不替代医生诊断

### 6. topic_products

用于连接 health_topics 和 products，支持按健康主题展示相关产品。

**关键字段：**

- `id`：唯一 ID
- `topic_id`：关联 health_topics.id
- `product_id`：关联 products.id
- `relevance_level`：相关程度，例如 high、medium、low
- `display_order`：页面展示排序
- `recommendation_note`：为什么该产品出现在此主题下
- `caution_note`：该主题下使用该产品的风险提醒

### 7. topic_ingredients（建议后续添加）

用于连接 health_topics 和 ingredients。这是科学逻辑层，比 topic_products 更重要。

**关键字段：**

- `id`：唯一 ID
- `topic_id`：关联 health_topics.id
- `ingredient_id`：关联 ingredients.id
- `relevance_level`：相关程度
- `evidence_level`：该成分与该主题的证据等级
- `explanation`：为什么该成分与该健康主题相关
- `caution_note`：风险提醒

### 8. ingredient_interactions（后续添加）

用于记录成分之间或成分与药物之间的相互作用。

**关键字段：**

- `id`：唯一 ID
- `ingredient_id`：主成分
- `interaction_target`：相互作用对象，例如 Warfarin、Iron、Calcium
- `target_type`：对象类型，例如 ingredient、drug、condition
- `interaction_level`：风险等级，例如 low、medium、high
- `description`：相互作用说明
- `recommendation`：建议处理方式

## 数据关系总结

**核心关系：**

```text
brands
↓
products
↓
product_ingredients
↓
ingredients
```

**健康主题关系：**

```text
health_topics
↓
topic_products
↓
products
```

**更推荐的科学逻辑：**

```text
health_topics
↓
topic_ingredients
↓
ingredients
↓
product_ingredients
↓
products
```

平台长期目标是从“产品库”升级为“德国药房式健康知识图谱”。
